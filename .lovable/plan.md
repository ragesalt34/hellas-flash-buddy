

# Генерация озвучки для всех вопросов на греческом

## Текущая ситуация

- 168 вопросов в базе, из них озвучено ~15-20 на греческом
- Функция `generate-all-tts` уже существует, но генерирует и RU и EL одновременно
- Функция `elevenlabs-tts` проверяет кэш — если файл уже есть, повторно не генерирует
- Edge function имеет таймаут ~60 сек, поэтому за один вызов можно обработать ~5-10 вопросов

## Изменения

### 1. `supabase/functions/generate-all-tts/index.ts` — добавить фильтр по языку

Добавить параметр `language` в запрос. Если `language = "el"`, генерировать только греческие файлы (`question_el`, `correct_answer_el`). Если не указан — генерировать все языки (текущее поведение).

### 2. Вызвать функцию последовательно

После деплоя вызвать функцию несколько раз с параметрами:
- `{ offset: 0, limit: 10, language: "el" }`
- `{ offset: 10, limit: 10, language: "el" }`
- ... и так далее до offset 170

Каждый вызов обработает 10 вопросов (до 20 аудиофайлов: вопрос + ответ на греческом). Функция `elevenlabs-tts` автоматически пропустит уже существующие файлы.

### Технические детали

```
generate-all-tts  -->  elevenlabs-tts (x2 per question)
                        |-> Check cache in tts-audio bucket
                        |-> If missing: call ElevenLabs API
                        |-> Save to tts-audio bucket
```

Ориентировочно потребуется ~17 последовательных вызовов (168 вопросов / 10 за раз). Общее время — около 15-20 минут с учётом задержек между запросами к ElevenLabs API.
