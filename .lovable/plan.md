
## Анализ badge-индикаторов на /learn

### Что такое badge-индикаторы на /learn
1. **AccBadge** — показывает точность (%) в правом верхнем углу карточки темы. Появляется только если `acc !== null` (то есть если пользователь хотя бы раз отвечал на вопросы этой темы).
2. **`.lp-due`** — бейдж «Сегодня: N карточек» в правом верхнем углу хедера страницы.

### Найденные проблемы

**Проблема 1 — AccBadge логика цвета сломана:**
```tsx
const hi = acc >= 70, mid = acc >= 50;
```
Если `acc >= 70`, то и `hi = true`, и `mid = true`. Но в рендере используется `hi ? ... : mid ? ... : ...` — это работает корректно для отображения, однако смысл `mid` неверен: он покрывает диапазон 50–69. Это нормально.

**НО**: на скриншоте видно badge `44%` с красным цветом (AlertCircle) — правильно. Badge `33%` — тоже красный — правильно. Badge `0%` — красный — **проблема**: если у темы вообще нет прогресса (`acc === null`), badge не показывается вовсе. Но `0%` — это значит пользователь отвечал и всегда ошибался. Это верно.

**Главная реальная проблема — badge не отображается на картах без прогресса:**
Строка 179: `{acc !== null && <AccBadge acc={acc} compact />}` — у тем без ни одного ответа badge скрыт. На скриншоте видно что у «Законов» вообще нет бейджа (правильно — нет данных). Но у «Географии» показывает `0%` — значит там есть хотя бы один ответ. Это логика верная.

**Проблема 2 — AccBadge показывает КУМУЛЯТИВНУЮ точность, а не "реальную" точность:**
В `topicProgress` считается:
```ts
stats[topic].correct += p.correct_count;
stats[topic].total += p.correct_count + p.incorrect_count;
```
`correct_count` и `incorrect_count` — это суммарные счётчики за всё время. Если пользователь раньше ошибался, а теперь улучшился — badge по-прежнему покажет старую низкую точность. Это не баг, но нужно осознавать.

**Проблема 3 (реальный баг) — `.lp-due` badge скрыт при `dueCount === 0`:**
```tsx
{!!dueCount && dueCount > 0 && (
```
`!!dueCount` — если `dueCount === 0`, это `false`. Если `dueCount === undefined` (запрос ещё грузится), это тоже `false`. Всё правильно — badge не мигает при загрузке и не показывает "0 карточек". Это норм.

**Проблема 4 (реальный баг) — `seen` vs `mastered` несоответствие:**
Кольцо прогресса (Ring) считает `mastered / total`, но по памяти: "Общий прогресс рассчитывается как процент *затронутых (seen)* карточек относительно общего числа". Проверю:

Строка 134: `const pct = total > 0 ? Math.round(((tp?.mastered || 0) / total) * 100) : 0;`

`tp?.mastered` — это `is_known` карточки. `total` — все вопросы в теме. Значит кольцо показывает **% освоенных из всех**, а не % виденных. Это соответствует подписи "X / Y освоено". Это правильно.

**Итог — реальных визуальных багов нет, но есть одно улучшение:**

**AccBadge отсутствует у тем без прогресса** — вместо пустоты можно показать нейтральный серый badge "—" или "Нет данных", чтобы правый верхний угол не был пустым для тем без активности.
