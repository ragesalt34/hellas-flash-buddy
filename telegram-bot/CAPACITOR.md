# iOS app (personal sideload) — Hellas Study

A native iOS shell around the existing Mini App (`telegram-bot/webapp`), built with
**Capacitor**, compiled in the cloud by **Codemagic**, and installed on your iPhone
with **Sideloadly** — no Apple Developer Program ($99) and no App Store review.

## How it works
- The React UI is bundled into the app (`webDir: dist`). It loads instantly offline;
  only data (questions, SRS, TTS) is fetched over the network from the bot.
- Outside Telegram there's no `initData`, so the app authenticates via the bot's
  **dev-auth** bypass: it sends `X-Dev-User-Id: <your Telegram id>` (baked in at build
  time via `VITE_DEV_USER_ID`). The bot must run with `ALLOW_DEV_AUTH=true`.
- Backend base URL is `VITE_API_BASE` (baked in at build time).

## ⚠️ Prerequisite: a STABLE backend URL
The app has the backend URL compiled in. The current Cloudflare **quick tunnel changes
its address on every restart**, so an app built against it breaks within ~a day. Set up
the **permanent** URL first (the pending domain + Cloudflare named tunnel — see
`miniapp_hosting_state` memory). Until then this app is only good for a quick test.

## ⚠️ Security note
`ALLOW_DEV_AUTH=true` on a public URL means anyone who knows the URL + any Telegram id
can read/write that user's progress. Acceptable for a single personal user on an obscure
URL; do NOT leave it on for a public/shared deployment.

## One-time setup
1. **Push the repo** to GitHub (codemagic.yaml, capacitor config, etc. must be on the remote).
2. **Codemagic** (codemagic.io): sign up (free), connect the GitHub repo, pick the
   `ios-sideload` workflow.
3. Set **VITE_API_BASE** (your stable bot URL, no trailing slash) and **VITE_DEV_USER_ID**
   (your numeric Telegram id) — in `codemagic.yaml` `environment.vars`, or as encrypted
   vars in the Codemagic UI (preferred).
4. On the bot: add `ALLOW_DEV_AUTH=true` to `telegram-bot/.env` and restart it.
5. Install **Sideloadly** on Windows (sideloadly.io) + Apple's iTunes/iCloud (for the USB driver).

## Each build / install
1. Run the `ios-sideload` workflow in Codemagic → download `HellasStudy-unsigned.ipa`.
2. Open **Sideloadly**, plug in the iPhone via USB, drag the `.ipa` in, enter your free
   Apple ID, click **Start**. (First time: on the phone, Settings → General → VPN &
   Device Management → trust your developer certificate.)
3. The app appears on the home screen. **Free Apple ID signatures expire after 7 days** —
   re-run Sideloadly weekly to refresh (the build itself doesn't need rebuilding unless
   the code changed).

## Notes
- The iOS native project (`telegram-bot/webapp/ios/`) is generated by Codemagic each build
  (`cap add ios`) and is git-ignored — no need to commit it or own a Mac.
- App id: `study.hellas.app`, name "Hellas Study" (edit in `webapp/capacitor.config.ts`).
- The correct-answer sound and all UI assets are bundled; they work offline. Only the
  question/vocab data and Greek TTS audio require the backend to be reachable.
